Developer Documentation

Getting Started

This guide will help you set up the development environment and contribute to the Windows CPU & GPU Temperature Monitor project.

Prerequisites

  • Python 3.8 or later
  • Git
  • Windows 10 or later
  • Visual Studio Code (recommended) or any IDE of your choice

Setup Development Environment

  1. Clone the repository: 
    git clone https://github.com/mhuzaifadev/windows-cpu-temp.git
cd windows-cpu-temp
  2. Create a virtual environment: 
    python -m venv venv
.\venv\Scripts\activate
  3. Install dependencies: 
    pip install -r requirements.txt

Project Structure

windows-cpu-temp/
├── assets/              # Application assets
│   ├── icon.ico        # Application icon
│   └── icon.png        # Application logo
├── docs/               # Documentation and website
│   ├── css/           # Website styles
│   ├── images/        # Website images
│   └── *.html         # Website pages
├── main.py            # Main application code
├── WinTmp.py          # Temperature monitoring module
├── requirements.txt   # Python dependencies
└── README.md         # Project documentation

Building the Application

To build the executable:

pyinstaller --onefile --windowed --icon=assets/icon.ico main.py

The executable will be created in the dist directory.

Contributing

Development Workflow

  1. Fork the repository
  2. Create a new branch for your feature: 
    git checkout -b feature/your-feature-name
  3. Make your changes
  4. Commit your changes: 
    git commit -m "Add your feature description"
  5. Push to your fork: 
    git push origin feature/your-feature-name
  6. Create a Pull Request

Coding Standards

  • Follow PEP 8 style guide for Python code
  • Use meaningful variable and function names
  • Add comments for complex logic
  • Write docstrings for functions and classes
  • Keep functions small and focused

Testing

Before submitting a pull request, ensure your code works correctly:

  1. Test the application on different Windows versions
  2. Verify temperature readings are accurate
  3. Check for any memory leaks
  4. Ensure the UI is responsive

Version Control

We follow semantic versioning (MAJOR.MINOR.PATCH):

  • MAJOR: Incompatible API changes
  • MINOR: Backwards-compatible functionality
  • PATCH: Backwards-compatible bug fixes
